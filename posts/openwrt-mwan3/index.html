    Tutorial Load-balancing Sederhana di OpenWrt

    11 December 2020 · KopiJahe

    Pada artikel ini, saya akan coba menjelaskan langkah-langkah yang diperlukan untuk load-balancing sederhana 2 koneksi di OpenWrt. Sebagai contoh, saya gunakan 2 koneksi ke jaringan @wifi.id.

    ⚠️ Penggunaan 2 atau lebih koneksi bisa menyebabkan koneksi tidak stabil, terutama untuk game online, diperlukan pengaturan load-balance lebih lanjut untuk membagi koneksi agar stabil. Pada artikel ini, saya tidak membahas masalah pisah jalur/traffic koneksi, hanya murni load-balance saja.

    Selama pengaturan load-balance berjalan, koneksi ke internet kemungkinan besar akan terputus.

    1. Buka koneksi SSH ke router menggunakan PuTTY (biasanya: 192.168.1.1)

    2. Pastikan dalam keadaan terkoneksi dengan internet, lalu ketikkan perintah ini untuk mengunduh dan memasang paket yang dibutuhkan untuk load-balance:

    opkg update && opkg install luci-app-mwan3

    1. Setelah paket selesai diunduh dan dipasang, buka pengaturan openwrt melalui browser (biasanya: 192.168.1.1)

    2. Buka menu Network -> Wireless

    3. Lakukan koneksi kedua seperti sebelumnya dengan menekan tombol Scan -> pilih pemancar

    4. Beri nama unik koneksi kedua, misalnya wwan2, lalu di bagian Create / Assign Firewall Zone, pilih custom, dan beri nama unik zona firewall baru, misalnya wanb, lalu tekan tombol Submit

    5. Setelah berhasil terkoneksi, buka menu Network -> Interfaces

    6. Tekan tombol Edit di bagian koneksi WWAN

    7. Buka tab Advanced Settings, lalu masukkan nilai 10 di bagian Use gateway metric, lalu tekan Save and Apply

    8. Ulangi langkah no. 8-9 untuk koneksi WWAN2, tapi masukkan nilai 20 di bagian Use gateway metric, lalu tekan Save and Apply

    1. Setelah itu, masuk ke menu Network -> Firewall, di sini kita akan meniru pengaturan bawaan zona firewall wan ke zona firewall baru wanb

    1. Tekan tombol edit di baris wanb, lalu masukkan parameter di bawah, lalu tekan tombol Save and Apply
    [*] Name: wanb
[*] Input: reject
[*] Output: accept
[*] Forward: reject
[*] Masquerading: <dicentang>
[*] MSS Clamping: <dicentang>
[*] Allow forward to destination zone: <kosong>
[*] Allow forward from source zone: lan

    1. Jika kita masuk ke tab Traffic Rules di menu pengaturan Firewall, di sana ada beberapa aturan lalu lintas data yang perlu digandakan

    1. Agar mudah, kita rubah via terminal SSH saja. Buka kembali koneksi SSH ke router menggunakan PuTTY (biasanya: 192.168.1.1)

    2. Ketikkan perintah ini untuk menampilkan konfigurasi firewall yang ada

    cat /etc/config/firewall

    1. Menggunakan mouse, blok semua bagian yang berawalan config rule, lalu gunakan klik kiri untuk menyimpan hasilnya

    1. Buka notepad, lalu paste hasil nomor 16 dengan menekan tombol Ctrl+V

    1. Tekan tombol Ctrl+H, lalu cari teks 'wan' dan ganti semua dengan 'wanb'

    1. Tekan lagi tombol Ctrl+H, lalu cari teks name ' dan ganti semua dengan name 'wanb-

    1. Kopikan hasil akhirnya dengan menekan tombol Ctrl+A diikuti tombol Ctrl+C

    1. Kembali ke jendela PuTTY, ketikkan perintah ini untuk mengedit berkas pengaturan firewall
    vi /etc/config/firewall

    1. Scroll ke baris terakhir dari berkas menggunakan tombol Page Down atau tombol arah bawah

    1. Untuk memulai mode input, tekan tombol i, perhatikan indikator di bagian pojok bawah kiri, akan berubah dari - menjadi I

    1. Paste hasil dari nomor 20 dengan klik kanan, lalu simpan hasilnya dengan menekan tombol ESC dan diikuti dengan mengetikkan :wq

    1. Agar perubahan bisa langsung diterapkan, ketikkan perintah ini, dan tunggu sampai proses selesai
    /etc/init.d/firewall restart

    1. Hasilnya bisa dilihat via browser di tab Traffic Rules halaman pengaturan OpenWrt -> Network -> Firewall seperti no. 13 tadi

    1. Setelah selesai mengatur firewall, masuk ke menu Network -> Load balacing

    1. Hapus semua baris di halaman Interfaces

    1. Isikan wwan pada kotak teks di bagian bawah, lalu tekan tombol Add

    1. Masukkan nilai-nilai berikut di halaman pengaturan interface, lalu tekan tombol Save (jangan Save and Apply, supaya tidak menunggu terlalu lama)
    [*] Enabled: <dicentang>
[*] Initial state: Online
[*] Internet Protocol: IPv4
[*] Tracking hostname or IP address: 8.8.8.8
                                     8.8.4.4
                                     1.1.1.1
                                     1.0.0.1
[*] Tracking method: ping
[*] Tracking reliability: 1
[*] Ping count: 3
[*] Ping size: 56
[*] Max TTL: 60
[*] Check link quality: <tidak dicentang>
[*] Ping timeout: 4 seconds
[*] Ping interval: 10 seconds
[*] Failure interval: 5 seconds
[*] Keep failure interval: <tidak dicentang>
[*] Recovery interval: 5 seconds
[*] Interface down: 5
[*] Interface up: 5
[*] Flush conntrack table: <tidak dicentang>

    1. Kembali ke halaman interface, lalu ulangi langkah no. 29 dan 30 untuk interface wwan2

    1. Setelah kedua interface selesai diatur, sekarang masuk ke tab Members, lalu hapus semua isinya

    1. Untuk isikan wwan_m1_w1 (yang berarti wwan metric 1 weight 1) pada kotak teksi di bagian bawah, lalu tekan tombol Add

    1. Pilih interface wwan dan isikan angka 1 di bagian metric dan weight, lalu tekan tombol Save

    1. Ulangi langkah no. 33 dan 34 untuk interface wwan2

    1. Setelah kedua member selesai diatur, sekarang masuk ke tab Policies, lalu hapus semua isinya kecuali baris balanced

    1. Tekan tombol Edit di baris balanced, lalu pilih kedua member yang baru kita buat di nomor 32-35

    1. Tekan tombol Save and Apply, dan pengaturan load-balance sudah selesai.
